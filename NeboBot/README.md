# Telegram-бот региональных оповещений о БПЛА

Рабочее название проекта — **BPLA Region Bot**. Папку и отображаемое имя бота можно
переименовать позже: логика от названия не зависит.

## Что умеет

- при первом `/start` предлагает выбрать регион из списка сайта или найти его по названию;
- хранит выбор и настройки уведомлений в локальной SQLite-базе;
- раз в минуту проверяет новые публикации и присылает подходящие выбранному региону;
- показывает по кнопке **«📊 Что сейчас»** статистику за последние 24 часа, разбивку по
  типам и пять последних записей;
- для каждого региона показывает **уровень риска**, **всего инцидентов** и
  **сработки/упоминания ПВО по публикациям**;
- строит общую сводку по России и показывает регионы, где последняя запись остаётся
  тревожной;
- предлагает два режима уведомлений: все события или только тревоги/обнаружения/ПВО и
  обязательные сообщения об отбое;
- географию рассылки можно переключать независимо от домашнего региона: только выбранный
  регион или вся Россия;
- кэширует популярные сводки, объединяет одновременные запросы и при кратком сбое может
  показать последний успешно рассчитанный результат;
- позволяет сменить регион, временно отключить уведомления и удалить свои данные.

Источник — открытый WordPress REST API сайта
[bplarussia.ru](https://bplarussia.ru/regions/). При первом запуске текущая лента только
запоминается и не рассылается, поэтому пользователи не получают пачку старых сообщений.
После перезапуска бот продолжает с последней сохранённой записи.

## Важное ограничение

Сайт публикует **сообщения об инцидентах**, а не телеметрию радара. Поэтому бот может
посчитать публикации, предупреждения, сообщения об обнаружении и упоминания ПВО, но не
может достоверно сообщить реальное число беспилотников. В интерфейсе это подписано явно.
Поле «Сработок ПВО» считается как число архивных публикаций региона, содержащих «ПВО»;
это воспроизводимый показатель публикаций, а не подтверждённое военное число. Уровень
риска повторяет историческую шкалу bplarussia.ru и также не является официальным прогнозом.
Бот не является официальной системой оповещения; экстренные указания нужно сверять с
сообщениями властей и службами 112.

## Дополнительные источники

В меню есть отдельный экран источников. После проверки добавлены ссылки на:

- [RadarMap](https://radar-map.ru/) — неофициальная карта публичной ленты Радар ВРВ;
- [Воздушную обстановку](https://bplaalert.ru/) — агрегатор нескольких публичных каналов
  с детализацией по районам;
- [Радар ВРВ](https://radarvrv.ru/) — публичную ленту мониторинга;
- [официальное приложение МЧС России](https://mchs.gov.ru/deyatelnost/informacionnye-sistemy/mobilnoe-prilozhenie-mchs-rossii).

Данные сторонних агрегаторов намеренно не смешиваются с основной статистикой: их
собственные условия называют информацию справочной и не гарантируют точность или доставку.
Так пользователь видит происхождение данных и может быстро перепроверить тревогу.

## Запуск на Windows

Откройте PowerShell в этой папке и выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
Copy-Item .env.example .env
```

Создайте нового бота через `@BotFather`, вставьте его токен в `BOT_TOKEN` файла `.env`,
затем запустите:

```powershell
.\.venv\Scripts\python.exe main.py
```

После первой установки можно пользоваться `start_bot.cmd`.

## Установка на Linux-сервер

Скопируйте папку проекта на сервер и запустите от `root`:

```bash
bash deploy/setup_server.sh
```

Установщик безопасно запросит токен без отображения, создаст отдельного системного
пользователя `nebo-alert`, приложение `/opt/nebo-alert-bot`, базу
`/var/lib/nebo-alert-bot/bot.sqlite3` и службу `nebo-alert-bot.service` с автоматическим
перезапуском. Проверка состояния:

```bash
systemctl status nebo-alert-bot.service
journalctl -u nebo-alert-bot.service -f
```

## Автоматическое обновление

Серверный `nebo-deploy-webhook.service` принимает подписанные события GitHub на
`http://SERVER_IP:9002/github`. После push в `main` он проверяет новую версию в отдельном
worktree, запускает тесты, применяет только fast-forward обновление и перезапускает бота.
Если служба не запускается, код автоматически откатывается. После успешной проверки
админ-бот отправляет владельцу сообщение «Небо рядом обновлён и работает».

Секрет webhook хранится только в `/etc/nebo-alert-bot.env` как `WEBHOOK_SECRET`; токен
админ-бота берётся из защищённого `/home/bot/.env` или из переменной `ADMIN_BOT_TOKEN`.

## Настройки `.env`

- `BOT_TOKEN` — обязательный токен Telegram-бота;
- `POLL_INTERVAL_SECONDS` — частота проверки сайта, минимум 30 секунд;
- `DATABASE_PATH` — путь к SQLite-базе;
- `STATS_CACHE_TTL_SECONDS` — срок кэша сводок, по умолчанию 30 секунд;
- `MAX_HISTORY_PAGES` — защита от слишком большой догрузки ленты после долгого простоя;
- `TELEGRAM_PROXY_URL` — необязательный прокси для Telegram Bot API.

## Проверка

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests -v
```

Тесты не обращаются к Telegram и не меняют данные других ботов.
