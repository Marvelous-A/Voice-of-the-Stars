# ЭХО | Анонимный чат

Telegram-бот для анонимных разговоров один на один: [@EchoDialogBot](https://t.me/EchoDialogBot).

## Что уже реализовано

- подтверждение возраста 18+;
- случайный подбор с взаимным фильтром пола;
- приоритет совпадающих интересов;
- анонимная передача текста, фото, видео, GIF, документов, стикеров,
  голосовых, видеосообщений и аудио;
- скрытие входящих фото, видео и GIF под спойлером;
- команды `/search`, `/next`, `/stop`, `/link`, `/interests`, `/settings`;
- общая ссылка на аудио- или видеозвонок для текущей пары через WebRTC;
- Premium и VIP с оплатой через CKassa, дневным лимитом бесплатных диалогов,
  платным фильтром пола, VIP-приоритетом и значком;
- оценки, жалобы и исключение пожаловавшихся пользователей из будущего подбора;
- полное удаление данных пользователя;
- команды модератора `/stats`, `/ban ID`, `/unban ID`;
- SQLite без хранения содержимого переписки.
- учёт первого рекламного источника по deep link и уведомления администраторам.
- возвращающее сообщение зарегистрированным пользователям раз в три дня в 21:00 МСК.

## Premium, VIP и CKassa

Тарифы: Premium на 7 дней — 149 ₽, на месяц — 249 ₽, на год — 999 ₽;
VIP на год — 3499 ₽. Premium открывает фильтр пола собеседника, звонки,
снимает дневной лимит и рекламу. VIP включает Premium и дополнительно даёт
приоритет в очереди и видимый собеседнику VIP-значок.

Свой пол, интересы, обычный поиск, сообщения, медиа, жалобы и открытие профиля
остаются бесплатными. Бесплатный дневной лимит задаётся через
`FREE_DAILY_DIALOG_LIMIT` (по умолчанию 15).

CKassa использует те же переменные `CKASSA_API_LOGIN`,
`CKASSA_API_AUTHORIZATION` и `CKASSA_SERV_CODE`, что и «Голос Звёзд».
На сервере служба сначала читает `/home/bot/.env`, затем
`/etc/echo-dialog-bot.env`; собственные настройки «ЭХО» имеют приоритет.
После изменения systemd-файла один раз повторно запустите `deploy/setup_server.sh`.

## Рекламные ссылки

Для каждого размещения используйте отдельную ссылку вида:

```text
https://t.me/EchoDialogBot?start=channel_name
```

Параметр источника может содержать латинские буквы, цифры, `_` и `-` (до 64 символов).
Первый источник пользователя сохраняется в базе, а администраторы из `ADMIN_IDS` получают
уведомление от отдельного админ-бота с токеном `ADMIN_BOT_TOKEN`. ЭХО такие уведомления
не отправляет. Сводка по источникам также показывается в команде `/stats`.

## Запуск на Windows

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python main.py
```

В текущей рабочей папке окружение и зависимости уже подготовлены. Для запуска
достаточно открыть `start_bot.cmd` или выполнить:

```powershell
.\.venv\Scripts\python.exe main.py
```

Токен читается из `.env`. Этот файл исключён из Git. Перед публикацией проекта
оставьте в репозитории только `.env.example`.

Чтобы получать жалобы и пользоваться командами модератора, узнайте свой
числовой Telegram ID и добавьте его в `.env`:

```dotenv
ADMIN_IDS=123456789
```

Несколько ID перечисляются через запятую.
Также можно задать `ADMIN_USERNAMES=bimbim2bambam`: бот найдёт числовой ID среди
пользователей, которые уже запускали его. Для надёжности рекомендуется оставить и `ADMIN_IDS`.

## Приватность

Содержимое сообщений не записывается в SQLite и не выводится в журнал.
Telegram-файлы передаются по `file_id`, остальные поддерживаемые сообщения —
через `copyMessage`, поэтому имя отправителя не показывается. Служебные данные
(профиль, очередь, факт диалога, оценки, жалобы и технические ID пар сообщений
для отображения ответов) сохраняются в `echo.db`.

Это не сквозное шифрование: сообщения технически проходят через Telegram Bot API.
Звонки по умолчанию открываются на `meet.jit.si`; этот внешний сервис обрабатывает
медиа и сетевые метаданные участников. Провайдера можно заменить через
`CALL_BASE_URL` или развернуть собственный Jitsi Meet.

## Развёртывание

На сервере бот запускается службой `echo-dialog-bot.service`. Отдельная служба
`echo-deploy-webhook.service` принимает подписанные события GitHub на пути
`/github`, проверяет код и тесты, а затем перезапускает бота. Серверные файлы
находятся в каталоге `deploy/`.

Каждый push в ветку `main` автоматически развёртывается на сервере после успешной проверки.
После подтверждённого обновления или ошибки администратор получает сообщение от отдельного
админ-бота, токен которого задаётся в `ADMIN_BOT_TOKEN`. Пользовательский бот «ЭХО» для
служебных уведомлений не используется. Администратор должен заранее открыть диалог с
админ-ботом и нажать «Старт».
